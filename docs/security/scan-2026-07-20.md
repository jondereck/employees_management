The highest-risk issue is that **Clerk middleware treats every `/api/*` route as public**, and many handlers do not perform their own authentication. Several endpoints allow unauthenticated data modification, enumeration of HR data, or abuse of paid third-party APIs.

---

## Critical — fix immediately

### 1. All API routes bypass Clerk middleware

```3:8:middleware.ts
export default authMiddleware({
  publicRoutes: [
    "/api/:path*",
    "/(.*)/view/employee/(.*)",   // printed QR URLs
  ],
});
```

**Impact:** Clerk session enforcement is disabled for the entire API surface. Protection depends entirely on each route calling `auth()` or a helper — and many do not.

**Fix:** Remove `/api/:path*` from `publicRoutes`. Whitelist only routes that must be public (e.g. `/api/public/*`, `/api/webhooks/*`, specific autofill endpoints). Add a shared `requireAuth()` helper used by all protected handlers.

---

### 2. Unauthenticated employee deletion

**Route:** `DELETE /api/[departmentId]/employees/delete`

No `auth()` check. Any caller who knows a `departmentId` and employee UUIDs can permanently delete records.

**Fix:** Require authentication and verify the user owns or administers the department before `deleteMany`.

---

### 3. Unauthenticated admin timeline writes

**Route:** `POST /api/admin/employees/[employeeId]/timeline`

Creates `employmentEvent` rows with no authentication or authorization.

**Fix:** Require admin auth (Clerk role metadata) and validate `employeeId` belongs to an allowed department.

---

### 4. Unauthenticated public-profile toggle

**Route:** `POST /api/employee/[employeeId]/toggle-public`

Flips `publicEnabled` for any employee ID without login.

**Fix:** Require authenticated department owner/admin; verify employee belongs to their department.

---

### 5. Dev/debug endpoints exposed in production

| Route | Risk |
|-------|------|
| `POST /api/dev/pusher-test` | Triggers Pusher events on `dev-hrps` channel |
| `GET /api/dev/public-check/[departmentId]/[employeeId]` | Reveals employee existence, department match, and PII fields |

**Fix:** Gate behind `NODE_ENV === "development"` or remove entirely from production builds.

---

### 6. Infrastructure admin endpoints are public

| Route | Risk |
|-------|------|
| `GET /api/neon/ping` | Uses `NEON_API_KEY` server-side; returns Neon API response body to any caller |
| `GET /api/neon/usage` | Exposes Neon org/project usage via server API key |

**Fix:** Protect with admin auth or remove from production. Never expose infra diagnostics publicly.

---

### 7. Public employee data without `publicEnabled` check

These routes return employee HR data by UUID without verifying the employee opted into public visibility:

- `GET /api/public/employees/[employeeId]/basics` — date hired, position, office, employee type
- `GET /api/public/employees/[employeeId]/timeline` — full employment events and awards

Other `/api/public/*` routes correctly check `publicEnabled: true`.

**Fix:** Add `publicEnabled: true` to the Prisma `where` clause on both routes, consistent with sibling public endpoints.

### 9. Debug response leaks database host

```489:495:app/api/import/attendance/resolve/route.ts
      debug: {
        idType: body.idType,
        idsRequested: ids.length,
        idsFound: employees.length,
        sampleMissing,
        dbUrlHost: process.env.DATABASE_URL?.split("@")[1]?.split("/")[0],
      },
```

**Impact:** Unauthenticated callers learn which database host the app connects to.

**Fix:** Remove `debug` from production responses; require auth on this route.

---

### 10. Weak Genio context signing secret fallback

```114:121:src/genio/context.ts
function contextSecret() {
  return (
    process.env.GENIO_CONTEXT_SECRET ||
    process.env.CLERK_SECRET_KEY ||
    process.env.OPENAI_API_KEY ||
    process.env.DATABASE_URL ||
    "genio-dev-context-secret"
  );
}
```

**Impact:** Predictable HMAC secret in dev; reusing production secrets for a different purpose increases blast radius if context tokens leak.

**Fix:** Require `GENIO_CONTEXT_SECRET` in production; fail fast if unset. Remove hardcoded fallback.

---

### 11. No HTTP security headers

`next.config.js` does not set CSP, `X-Frame-Options`, `HSTS`, `X-Content-Type-Options`, or `Referrer-Policy`.

**Fix:** Add a `headers()` export in `next.config.js` with baseline security headers.

---

### 12. Authorization is inconsistent

Many routes call `auth()` and check `userId` but **do not verify department ownership**. Example pattern risk: any logged-in Clerk user may access any `departmentId` if the handler only checks `userId` presence.

**Fix:** Introduce a shared `requireDepartmentAccess(departmentId)` used across department-scoped routes (similar to `requireBackupAccess` / `requireOrgChartDepartmentAccess`).
